AgentCore Memory vs DynamoDB for agent state

When you build an agent that needs to remember things between turns, the first decision is where that state lives. AWS gives you two natural homes in ap-southeast-2: managed AgentCore Memory, or a DynamoDB table you run yourself.

MIKE: Personal anecdote - your first AgentCore deployment in Sydney

The trade-off comes down to how much of the retrieval and summarisation you want to own. AgentCore Memory handles session and long-term recall for you; DynamoDB hands you the primitives and the bill.
